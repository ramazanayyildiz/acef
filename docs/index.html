Why ACEF exists

Generic workflows break when the project context is missing.

Wrong local pattern

The agent writes plausible code, but not the way this repo works.

Too much ceremony

Small fixes get pulled through heavyweight planning steps they do not need.

Stack assumptions

"Web" or "mobile" is not enough. Each repo can have different tooling and conventions.

Unclear proof

Documents, skills, and working evidence get mixed together unless the system labels them honestly.

The model

ACEF separates universal method from project-specific truth.

Agent behavior

The user should not need to know the system internals.

Guide

Names the current route and the next useful step in one or two sentences.

Route

Asks at most a few clarifying questions, then chooses the safest matching path.

Coordinate

Uses adapter, specify, test-bootstrap, and release helpers only when the route needs them.

Teach

Explains ACEF on demand: short by default, deeper when the user asks.
